# 03 — Codegen + legacy CLI

Authoring: SymEngine DSL in `pdemodel.txt`. text2code reads it and
emits `my_model.hpp` plus `libpdemodel{serial,cuda,hip}.{so,dylib}`.
Driving: same three-line `main.cpp` as path 01, but with
`GeneratedModel` instead of `Poisson2D`:

```cpp
#include <exasim/run.hpp>
#include "my_model.hpp"
int main(int argc, char** argv) {
    return exasim::run<GeneratedModel>(argc, argv);
}
```

Source: `apps/library_example/poisson2d_codegen/`. The
`pdemodel.txt` is the symbolic spec; `main.cpp` is generated by
`port_codegen.sh`. After `regenerate.sh`, `my_model.hpp` is
generated.

## Generate + run

```bash
cd apps/library_example/poisson2d_codegen
bash ../regenerate.sh poisson2d              # writes my_model.hpp + libpdemodel*.dylib
cmake --build $EXASIM/build_cpu --target poisson2d_codegen
$EXASIM/build_cpu/poisson2d_codegen ./pdeapp.txt
```

For MPI / GPU / MPI+GPU substitute the matching binary suffix and
build dir.

## What text2code generated

`apps/library_example/poisson2d_codegen/my_model.hpp` defines the
same `Poisson2D` struct shape that
[`01-handwritten-cli/`](../01-handwritten-cli/README.md) wrote by
hand. Both satisfy the contract in
[`../../docs/03-methods/model-contract.md`](../../docs/03-methods/model-contract.md).
